## 문서 개요

본 문서는 Briefly 플랫폼의 주요 유즈케이스를 정리한 명세서입니다. 사용자 관점에서 시스템과의 상호작용을 체계적으로 분석하고, 각 기능의 상세한 플로우를 정의합니다.

### 문서 목적
- 시스템 요구사항의 명확한 정의
- 개발팀과 이해관계자 간의 공통 이해 기반 마련
- 테스트 케이스 작성을 위한 기준 제공
- 향후 기능 확장을 위한 가이드라인 제시

## 1. 사용자 인증 및 관리

### UC-01: 카카오 소셜 로그인
**액터**: 신규/기존 사용자  
**목표**: 카카오 계정을 통한 간편 로그인  

**기본 플로우**:
1. 사용자가 "카카오로 시작하기" 버튼 클릭
2. 카카오 인증 서버로 리다이렉트
3. 사용자가 카카오 계정으로 로그인
4. 인증 코드와 함께 콜백 URL로 복귀
5. 백엔드에서 액세스 토큰 발급
6. 사용자 정보 조회 및 JWT 토큰 생성
7. 프론트엔드에 JWT 토큰 반환
8. 로컬 스토리지에 토큰 저장 후 메인 페이지로 이동

**대체 플로우**:
- A1: 카카오 인증 실패 시 → 에러 메시지 표시 후 로그인 페이지로 복귀
- A2: 네트워크 오류 시 → 재시도 옵션 제공

### UC-02: 사용자 프로필 관리
**액터**: 인증된 사용자  
**목표**: 개인 정보 조회 및 수정  

**기본 플로우**:
1. 사용자가 프로필 페이지 접속
2. JWT 토큰 검증
3. 사용자 정보 조회 (닉네임, 이메일, 프로필 이미지)
4. 프로필 정보 화면에 표시
5. 사용자가 정보 수정 시 → 업데이트 API 호출
6. 성공 메시지 표시

### UC-03: 관심 카테고리 설정
**액터**: 인증된 사용자  
**목표**: 개인화 서비스를 위한 관심 카테고리 선택  

**기본 플로우**:
1. 사용자가 카테고리 설정 페이지 접속
2. 6개 카테고리 목록 표시 (정치, 경제, 사회, 생활/문화, IT/과학, 연예)
3. 사용자가 관심 카테고리 선택/해제
4. "저장" 버튼 클릭
5. 백엔드에 선택된 카테고리 저장
6. 확인 메시지 표시 후 개인화된 메인 페이지로 이동

## 2. 뉴스 및 콘텐츠

### UC-04: 카테고리별 뉴스 조회
**액터**: 모든 사용자 (인증 불필요)  
**목표**: 특정 카테고리의 최신 뉴스 목록 확인  

**기본 플로우**:
1. 사용자가 카테고리 탭 선택
2. 해당 카테고리의 뉴스 목록 API 호출
3. 뉴스 카드 형태로 목록 표시 (제목, 썸네일, 발행시간)
4. 무한 스크롤로 추가 뉴스 로드

### UC-05: 뉴스 상세 보기
**액터**: 모든 사용자  
**목표**: 특정 뉴스의 전체 내용 확인  

**기본 플로우**:
1. 사용자가 뉴스 카드 클릭
2. 뉴스 상세 정보 API 호출
3. 상세 페이지 표시 (제목, 본문, 이미지, 발행정보)
4. 북마크 버튼 제공 (인증된 사용자만)
5. 관련 뉴스 추천 표시

### UC-06: 북마크 관리
**액터**: 인증된 사용자  
**목표**: 관심 뉴스 저장 및 관리  

**기본 플로우**:
1. 사용자가 뉴스 상세 페이지에서 북마크 버튼 클릭
2. 북마크 추가/제거 API 호출
3. 버튼 상태 변경 (북마크됨/안됨)
4. 내 북마크 페이지에서 저장된 뉴스 목록 확인 가능

## 3. 팟캐스트 및 주파수

### UC-07: 개인 맞춤 주파수 재생
**액터**: 인증된 사용자  
**목표**: 관심 카테고리 기반 개인화된 팟캐스트 청취  

**기본 플로우**:
1. 사용자가 메인 페이지의 "내 주파수" 접속
2. 설정된 관심 카테고리 기반 주파수 목록 조회
3. 각 카테고리별 최신 팟캐스트 표시
4. 사용자가 재생 버튼 클릭
5. S3에서 오디오 파일 스트리밍 시작
6. 재생 컨트롤 (재생/일시정지, 탐색바, 속도 조절) 제공

### UC-08: 카테고리별 주파수 청취
**액터**: 모든 사용자  
**목표**: 특정 카테고리의 팟캐스트 청취  

**기본 플로우**:
1. 사용자가 특정 카테고리 페이지 접속
2. 해당 카테고리의 주파수 정보 조회
3. 팟캐스트 정보 표시 (제목, 대본 미리보기, 재생 시간)
4. 재생 버튼 클릭 시 오디오 스트리밍 시작
5. 대본 전체 보기 옵션 제공

### UC-09: 주파수 히스토리 조회
**액터**: 인증된 사용자  
**목표**: 이전에 청취한 팟캐스트 목록 확인  

**기본 플로우**:
1. 사용자가 "히스토리" 페이지 접속
2. 날짜별 주파수 목록 조회
3. 카테고리별 필터링 옵션 제공
4. 각 항목에서 재청취 가능

## 자동화 시스템

### UC-10: 자동 뉴스 수집
**액터**: 시스템 (매일 오전 6시)  
**목표**: 카테고리별 최신 뉴스 자동 수집  

**기본 플로우**:
1. AWS EventBridge가 매일 오전 6시 Lambda 트리거
2. 6개 카테고리에 대해 병렬로 뉴스 수집 시작
3. 각 카테고리별로 정확히 30개 기사 수집
4. 부족한 본문은 재추출 수행
5. 수집된 뉴스를 DynamoDB에 저장
6. 수집 결과 로그 기록

### UC-11: 자동 팟캐스트 생성
**액터**: 시스템  
**목표**: 수집된 뉴스를 바탕으로 팟캐스트 자동 생성  

**기본 플로우**:
1. 뉴스 수집 완료 후 대본 생성 프로세스 시작
2. 카테고리별로 이중 클러스터링 수행
   - 1차: 원본 기사 물리적 중복 제거
   - 2차: GPT 요약문 의미적 중복 제거
3. GPT-4o-mini로 1800-2500자 대본 생성
4. ElevenLabs TTS로 음성 변환
5. S3에 오디오 파일 업로드
6. Presigned URL 생성 (7일 유효)
7. Frequencies 테이블에 결과 저장

### UC-12: 시스템 모니터링
**액터**: 시스템 관리자  
**목표**: 자동화 프로세스 상태 모니터링 및 장애 대응  

**기본 플로우**:
1. 각 처리 단계별 로그 기록
2. 성공/실패/스킵 결과 분류
3. 실패 시 자동 재시도 (최대 3회)
4. 심각한 오류 시 관리자 알림
5. 일일 처리 결과 요약 리포트 생성
